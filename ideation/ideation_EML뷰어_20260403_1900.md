# Ideation — EML 이메일 뷰어 기능

## 요청
`.eml` 파일 클릭 시 프로그램 내에서 이메일 내용(보낸 사람, 받는 사람, 제목, 본문)을 확인하고, 첨부파일도 열어볼 수 있는 기능.

---

## 1. 사전 조사

### 현재 동작
- 파일 더블클릭 → `shell.openPath()` → 시스템 기본 앱(Outlook 등)에서 열림
- 앱 내에서 파일 내용을 미리보기하는 기능 없음
- DetailsPane(우측 패널)에는 파일 메타데이터(이름, 크기, 날짜)만 표시

### EML 파일 구조
- RFC 2822/MIME 표준 텍스트 포맷
- 헤더(From, To, Subject, Date), 본문(text/plain, text/html), 첨부파일(MIME multipart)
- 한국어 이메일: EUC-KR/UTF-8 인코딩 혼재
- 일반적 크기: 100KB ~ 10MB (대형 첨부 시 더 클 수 있음)

## 2. 구현 방안 비교

| 접근법 | 설명 | 장점 | 단점 |
|--------|------|------|------|
| **A. DetailsPane 확장** | 우측 패널에 이메일 내용 + 첨부 목록 표시 | 기존 UI 흐름과 일관, 파일 선택만으로 미리보기 | 패널 폭 제한(200~500px), 긴 HTML 이메일 보기 불편 |
| **B. 전용 뷰어 컴포넌트** | FileList 영역을 대체하는 전용 이메일 뷰어 | 넓은 공간 활용, 풍부한 UI | 기존 탐색 흐름 중단, 별도 뒤로가기 필요 |
| **C. 모달 팝업** | 파일 클릭 시 중앙 모달로 이메일 표시 | 독립적 UI, 구현 간단 | 탐색 흐름 차단, UX 불편 |

### 추천: **A + B 혼합** — DetailsPane에서 미리보기 + 클릭 시 넓은 뷰어 전환
- **1차 구현**: DetailsPane 확장 (간단, 빠른 미리보기)
- **향후**: FileList 영역에서 전용 뷰어 옵션 추가

## 3. 제약사항

| 제약 | 영향 | 극복 방안 |
|------|------|----------|
| HTML 이메일 보안 (스크립트 실행 위험) | XSS 가능성 | `<iframe sandbox="allow-same-origin">` 사용 (스크립트 차단) |
| 인라인 이미지 (cid: 참조) | 깨진 이미지 표시 | 1차: 무시, 2차: cid → data URI 변환 |
| EUC-KR 인코딩 | 한국어 깨짐 | mailparser가 iconv-lite로 자동 처리 |
| .msg 파일 (Outlook 전용) | 바이너리 포맷, 파싱 어려움 | .eml만 지원, .msg는 외부 앱 안내 |
| 대용량 첨부파일 | 메모리 부담 | 파싱 시 메타데이터만 전달, 바이너리는 요청 시 추출 |

## 4. 최적 언어 선정

| 항목 | 판단 |
|------|------|
| 언어 | **TypeScript (기존 프로젝트)** — 신규 언어 도입 불필요 |
| Windows 11 빌드/실행 | ✅ Electron 기존 환경 |
| 인스톨러 패키징 | ✅ electron-builder 기존 설정 |
| 핵심 라이브러리 Windows 호환 | ✅ mailparser — Node.js 순수 JS, OS 무관 |

## 5. 핵심 라이브러리

| 패키지 | 용도 | 주간 다운로드 | Windows 호환 |
|--------|------|-------------|-------------|
| **mailparser** (v3.7+) | EML 파싱 (헤더, 본문, 첨부) | ~1.2M | ✅ |
| (내장) iconv-lite | 문자 인코딩 변환 | mailparser 번들 | ✅ |
