   - 키워드 저장 시 `dsff-custom-rules.json` + 원본 MD 파일 동시 업데이트
   - MD 파일 내 해당 폴더 섹션 키워드 줄 교체 + 상단 총 키워드 수 자동 갱신

2. **키워드 저장 후 UI 덮어쓰기 버그 수정**
   - 원인: reapply 콜백에서 원본 MD 재읽기 → `onAiStructure()` 호출이 App.tsx의 `setAiStructure` 업데이트를 덮어씀
   - 수정: reapply에서 MD 재읽기 + 트리 재빌드 코드 제거 (파일 재분류만 수행)

3. **탐색기 P0 기능 대량 구현** (파일/디렉토리 조작 전면 개선)

   #### 다중 파일 선택
   - `selectedFile: FileItem` → `selectedFiles: FileItem[]` 상태 변경
   - Ctrl+클릭 (토글), Shift+클릭 (범위), Ctrl+A (전체) 지원
   - StatusBar: 다중 선택 시 "N개 선택됨 (합산 크기)" 표시

   #### 클립보드 복사/잘라내기/붙여넣기
   - `clipboard: { mode: "copy"|"cut", files: FileItem[] }` 상태 추가
   - `fs:copyTo` IPC 핸들러 — 대상 경로에 파일 복사 (충돌 시 `파일(1).ext` 자동 처리)
   - `fs:moveFile` IPC 핸들러 — 대상 경로로 파일 이동
   - 컨텍스트 메뉴 + Ctrl+C/X/V 키보드 단축키 모두 지원
   - 잘라내기는 1회용 (붙여넣기 후 클립보드 자동 해제)
   - StatusBar에 클립보드 상태 표시 ("📋 복사 N개" / "📋 잘라내기 N개")

   #### 새 폴더 만들기
   - `handleNewFolder()` — 현재 경로에 "새 폴더" 생성 + 자동 이름 편집 모드 진입
   - 기존 `fs:createFolders` IPC 핸들러 활용 (신규 백엔드 불필요)
   - 컨텍스트 메뉴 (빈 영역 우클릭) + Ctrl+Shift+N 단축키

   #### 삭제 확인 대화상자
   - `deleteConfirm` 상태 + 확인/취소 모달
   - 단일: `"파일명"을(를) 삭제대상으로 이동하시겠습니까?`
   - 다중: `N개 항목을 삭제대상으로 이동하시겠습니까?` + 파일 목록

   #### FileList 인라인 이름 편집
   - `renamingFile` 상태 + FileList 내 인라인 input
   - F2 키 / 컨텍스트 메뉴 → 파일 목록에서 직접 이름 편집
   - Enter 확인, Escape 취소, 블러 시 자동 커밋
   - 확장자 앞까지만 자동 선택 (파일인 경우)

   #### 빈 영역 우클릭 메뉴
   - 파일 위: 열기/이름바꾸기/복사/잘라내기/이동/삭제/속성
   - 빈 영역: 새 폴더/붙여넣기(클립보드 있을 때)/새로고침

   #### 키보드 단축키 (13개)
   | 단축키 | 동작 |
   |--------|------|
   | Ctrl+C | 복사 |
   | Ctrl+X | 잘라내기 |
   | Ctrl+V | 붙여넣기 |
   | Ctrl+Z | 되돌리기 |
   | Ctrl+A | 전체 선택 |
   | Ctrl+Shift+N | 새 폴더 |
   | Delete | 삭제 (확인 포함) |
   | F2 | 이름 바꾸기 |
   | F5 | 새로고침 |
   | Enter | 열기 |
   | Backspace | 상위 폴더 |
   | ↑↓ | 파일 이동 |
   | Alt+←/→ | 뒤로/앞으로 |

### 결정 사항:
- 복사/이동 IPC는 `fs:copyTo`, `fs:moveFile`로 신규 분리 (기존 `fs:copy`는 같은 폴더 복사용으로 유지)
- 삭제는 항상 "삭제대상" 폴더 소프트 삭제 유지 (영구 삭제는 P2)
- `selectedFile` 단일 상태 → `selectedFiles[]` 배열로 전환, 하위 호환은 getter로 처리
- `ContextMenuState.file`을 nullable로 변경하여 빈 영역 우클릭 지원
- 키보드 단축키는 `window.addEventListener("keydown")`로 전역 등록 (input/textarea 제외)

### 작업 내용:

#### 1. UI 개선 3건
- **관리폴더 경로 컬러 구분**: AddressBar breadcrumb에서 관리폴더에 해당하는 세그먼트를 파란색(`#2563eb`)으로 표시
- **"검색" → "파일 이름 필터" 변경**: AddressBar 검색 기능명 + 아이콘을 필터 깔때기로 교체, 필터 활성 시 노란 테두리
- **파일 검색 기능 추가**: CommandBar에 "🔎 파일 검색" 토글 버튼 + 하위폴더 포함 재귀 검색, 결과 패널에 6컬럼(파일명/수정/만든/액세스/크기/경로) 표시

#### 2. 버그 수정 3건
- **FileList 레이아웃 깨짐**: `<td style="display:flex">` → `<td><div style="display:flex">` (테이블 행 높이 폭발 수정)
- **검색 결과 클릭 시 행 간격 깨짐**: `<table>` `minHeight: "100%"` 제거 (파일 적을 때 행 간격 비정상 확장)
- **NavigationPane 폴더 구분**: 8색 팔레트 배경 적용 + 폴더 간 구분선

#### 3. 필터/검색 탭 간 동작 개선
- **탭 전환 시 초기화**: `handleSetView()`에서 `searchQuery`, `fileSearchResults`, `fileSearchQuery` 초기화 + `filterResetKey` 증가
- **자식 컴포넌트 초기화**: AddressBar·CommandBar에 `resetKey` prop 전달 → useEffect로 로컬 state 초기화
- **비활성 탭 UI 숨김**: 필터/검색 UI를 "폴더 즐겨찾기"+"스마트 정리" 탭에서만 표시
- **스마트 정리 필터 연동**: SmartOrganizeView에 `searchQuery` prop 전달 → `filteredPlans` 필터링 + 필터 상태 표시

#### 4. 필터 입력 시 화면 빈 화면 크래시 수정
- **ErrorBoundary 추가**: React 렌더링 오류 시 회색 빈 화면 대신 오류 메시지 + "다시 시도" 버튼 표시
- **방어 코드**: `filteredFiles`·`filteredPlans` 필터링에 `f.name ?? ""` null 방어
- **필터 input 이벤트 격리**: `onKeyDown stopPropagation`으로 전역 키보드 핸들러 간섭 차단
- **breadcrumb commitEdit 강화**: 절대 경로 검증 (`/^[A-Za-z]:\\/` 또는 UNC) 추가, 잘못된 경로로 네비게이션 차단
- **AddressBar focused 상태 분리**: breadcrumb과 필터 간 `focused` 공유 제거

#### 5. 마지막 탐색 상태 복원
- **저장**: `%APPDATA%\dsff-webapp\dsff-last-state.json`에 `{folderPath, currentPath, selectedFile}` 자동 저장
- **복원**: 앱 시작 시 마지막 관리폴더 → 탐색 경로 → 선택 파일 순서로 복원
- IPC: `state:save` / `state:load` 핸들러 추가

#### 6. UI 브랜딩 및 아이콘
- **StatusBar 하단 중앙**: "신한DS AX본부" 텍스트 (연한 회색, 10px)
- **브랜딩 텍스트 변경**: "DS FolderFit" → "FolderFit powered by arti" (9px, 매우 연하게)
- **프로그램 아이콘**: 파란 배경 + 노란 폴더 + 체크마크 SVG → ICO/PNG 생성
  - 타이틀바: 이모지 → 실제 아이콘 이미지
  - 작업표시줄/EXE: `icon.ico` 적용 (main.cjs + package.json)
