# 스펙 — EML 이메일 뷰어

## 개요
DS FolderFit 앱에서 `.eml` 파일 선택 시 이메일 내용을 앱 내에서 직접 확인하고, 첨부파일을 열어볼 수 있는 기능.

---

## 요구사항 (EARS 원칙)

### Ubiquitous (항상 적용)
- **[U1]** The system SHALL display `.eml` files with a 📧 icon and "이메일 파일" type label in the file list.
- **[U2]** The system SHALL parse EML files using the `mailparser` library in the Electron main process.

### Event-driven (이벤트 기반)
- **[E1]** WHEN a user selects an `.eml` file in the FileList, THE system SHALL display the parsed email content (보낸 사람, 받는 사람, 참조, 제목, 날짜, 본문) in the DetailsPane.
- **[E2]** WHEN a user clicks an attachment's "열기" button, THE system SHALL extract the attachment to a temporary directory and open it with the system default application.
- **[E3]** WHEN a user double-clicks an `.eml` file, THE system SHALL open the file with the system default email client (기존 동작 유지).
- **[E4]** WHEN the application exits, THE system SHALL clean up the temporary attachment directory.

### Unwanted (예외 처리)
- **[X1]** IF the `.eml` file cannot be parsed (손상된 파일), THEN THE system SHALL display an error message "이메일 파일을 읽을 수 없습니다" with the error detail.
- **[X2]** IF the selected file has a `.msg` extension, THEN THE system SHALL display "Outlook MSG 형식은 지원되지 않습니다" with a button to open in the external app.
- **[X3]** IF an attachment extraction fails, THEN THE system SHALL display an error message with the attachment filename and error reason.
- **[X4]** IF the email body is empty (text and HTML both absent), THEN THE system SHALL display "(본문 없음)".

### State-driven (상태 기반)
- **[S1]** WHILE the EML file is being parsed, THE system SHALL display a loading indicator in the DetailsPane.
- **[S2]** WHILE the DetailsPane shows an email preview, THE system SHALL continue to display the standard file metadata (이름, 크기, 날짜) above the email content.

### Optional (조건부)
- **[O1]** WHERE the email contains HTML body, THE system SHALL render it in a sandboxed iframe (스크립트 실행 차단).
- **[O2]** WHERE the email contains only plain text body, THE system SHALL render it in a `<pre>` element with word wrap.
- **[O3]** WHERE the email has a CC field, THE system SHALL display it in the header section.

## 데이터 구조

### EmlData (파싱 결과)
```typescript
interface EmlAttachmentMeta {
  filename: string;       // 첨부파일명
  contentType: string;    // MIME 타입 (예: "application/pdf")
  size: number;           // 바이트 크기
  index: number;          // 첨부파일 인덱스 (추출 시 사용)
}

interface EmlData {
  from: string;           // 보낸 사람 (이름 <email>)
  to: string;             // 받는 사람
  cc: string;             // 참조 (없으면 빈 문자열)
  subject: string;        // 제목
  date: string;           // 날짜 (포맷: YYYY.MM.DD HH:mm)
  textBody: string;       // 텍스트 본문
  htmlBody: string;       // HTML 본문
  attachments: EmlAttachmentMeta[];
}
```

### IPC 채널
| 채널 | 방향 | 파라미터 | 반환 |
|------|------|---------|------|
| `eml:parse` | renderer → main | `filePath: string` | `DsffResult<EmlData>` |
| `eml:extractAttachment` | renderer → main | `filePath: string, attachIdx: number` | `DsffResult<{ path: string }>` |

## 보안 고려사항
- HTML 이메일: `<iframe sandbox="allow-same-origin">` — 스크립트 실행 완전 차단
- 첨부파일: 임시 폴더 추출 후 `shell.openPath` (OS 보안 정책 따름)
- 임시 파일: 앱 종료 시 자동 삭제 (`app.on("before-quit")`)
- 에러 메시지: 내부 경로/스택 트레이스 노출 금지
