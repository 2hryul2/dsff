# DS FolderFit 개발노트 — GUI-CLI 백엔드 연동

- **작성일**: 2026-03-28
- **작성자**: Ryul
- **버전**: v1.1.0 (예정)

## 작업 목표

관리폴더에서 대상 폴더 선택 시 6개 핵심 기능이 실제로 동작하도록
Electron GUI ↔ Python CLI 백엔드 IPC 브릿지를 구축한다.

## 작업 범위

### 1. Python CLI `--json` 출력 모드 (src/dsff/cli.py)
- `@app.callback()`에 `--json` 글로벌 옵션 추가
- analyze, organize, rename, duplicates, undo 커맨드에 JSON 분기
- 출력 형식은 기존 TypeScript 인터페이스(types.ts)와 정확히 매칭

### 2. Electron IPC 브릿지 (webapp/electron/main.cjs)
- `runDsff()` 범용 CLI 실행기 (child_process.spawn)
- 7개 IPC 핸들러: dsff:analyze, dsff:organize, dsff:rename, dsff:duplicates, dsff:undo, dsff:watchStart, dsff:watchStop
- Watch 모드 장기실행 subprocess 생명주기 관리

### 3. Preload 브릿지 (webapp/electron/preload.cjs)
- 8개 API 메서드 renderer에 노출

### 4. TypeScript 타입 선언 (webapp/src/)
- electron.d.ts: 새 API 타입
- types.ts: DsffResult, OrganizePlan, OrganizeResult, RenameResult, UndoResult, WatchEvent 등

### 5. App.tsx 상태 관리 리팩토링
- mock 데이터 의존성 제거
- 뷰 전환 시 자동 CLI 호출
- 로딩/에러 상태 관리

### 6. View 컴포넌트 업데이트 (4개)
- AnalyzeView: 실데이터 + 새로고침 + 추천작업 연동
- PreviewView: CLI dry-run 결과 표시 + 실행
- RenameView: 포맷 변경 시 실시간 미리보기 + 실행
- DuplicatesView: 해시 기반 실데이터 + 삭제/이동 처리

## 기술 결정 사항

1. **CLI 호출 방식**: `child_process.spawn("python", ["-m", "dsff", ...])` — 기존 CLI 코드 재사용 극대화
2. **데이터 포맷**: Python JSON 출력을 기존 TS 인터페이스에 맞춤 — 프론트엔드 변경 최소화
3. **Watch 모드**: ON/OFF 토글만 (이벤트 UI 없음), NDJSON 스트림은 콘솔에만 기록
4. **에러 처리**: `{ ok, data, error }` envelope 패턴 통일
5. **작업 완료 알림**: 모달 다이얼로그 (확인 버튼 필요)

## 테스트 계획

1. CLI: `python -m dsff analyze <path> --json` 직접 호출하여 JSON 파싱 확인
2. IPC: Electron dev 모드에서 각 기능 버튼 동작 확인
3. 에러: Python 미설치, 빈 폴더, 타임아웃 시 UX 확인
