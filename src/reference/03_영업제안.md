# 영업·제안 — 업무 PC 파일 정리 규칙

> **적용 환경** : 폐쇄망 · 사내 업무 포털(그룹웨어) · 사내 메신저·메일 파일 첨부 전달 · NAS / SVN 공유 드라이브

---

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 숫자 접두사 정렬 | `00_` → `01_` → `20_` 순으로 탐색기 정렬 고정 |
| 업무보고 최우선 | `00_업무보고`가 항상 최상단 노출 |
| PRE 코드 즉시 부여 | RFP 수신 즉시 기회 코드 부여 후 폴더 생성 |
| 발송본 별도 보관 | 메일로 발송한 제안서는 발송본 폴더에 반드시 보존 |

---

## 파일 전달 흐름

```
메일 RFP 수신 → 01_수신함 임시 보관 → 20_수주활동 PRE 폴더 생성
→ 제안서 작성 (NAS) → 메일 첨부 발송 + 22_제안서_발송본 보관
```

---

## 폴더 구조

```
📁 영업_제안
  📁 00_업무보고                           ← 최우선 노출
    📁 주간보고
    📁 월간보고
    📁 임원회의
  📁 01_수신함                             ← RFP·고객자료 메일 첨부 임시
  📁 20_수주활동
    📁 PRE-001_스마트팩토리
      📄 250501_RFP_수신.pdf
      📁 21_제안서_작업                    ← v1, v2, 최종
      📁 22_제안서_발송본                  ← 메일 첨부 발송본 원본
      📁 23_견적·원가
      📁 24_PT자료
    📁 PRE-002_ERP고도화
      ...
  📁 30_참조자료
  📁 90_Archive_실패                       ← 수주 실패 건 이동
```

## 파일명 규칙

| 유형 | 규칙 | 예시 |
|------|------|------|
| RFP 수신 | `YYMMDD_RFP_수신` | `250501_RFP_수신.pdf` |
| 제안서 작업본 | `YYMMDD_제안서_v번호` | `250510_제안서_v2.pptx` |
| 제안서 발송본 | `YYMMDD_제안서_발송` | `250515_제안서_발송.pptx` |
| 보고서 | `YYMMDD_보고명_상태` | `250603_주간영업현황_확정.xlsx` |

---

## 운영 규칙

- **`01_수신함`** : RFP·고객 자료 메일 첨부파일을 임시 보관하고, 당일 `PRE-번호` 폴더로 이동해 `YYMMDD_내용` 형식으로 재명명한다.
- **발송본 보관 원칙** : 고객사에 메일로 제안서를 발송한 후 `22_제안서_발송본`에 동일 파일을 저장한다. 고객사 수정 요청 시 어떤 버전을 발송했는지 즉시 확인할 수 있다.
- **수주 인계** : 수주 성공 시 `PRE-번호` 폴더를 `PRJ-코드`로 이름을 바꿔 개발 PM에게 NAS 경로를 메신저·메일로 인계한다.
- **수주 실패** : 실패 확정 시 해당 `PRE-번호` 폴더 전체를 `90_Archive_실패`로 이동해 활성 수주 목록을 정리한다.
- **원가·견적 보안** : 원가·견적 파일은 메신저 전달 금지. 반드시 영업팀 전용 NAS 경로를 공유하거나 암호화 메일로 발송한다.
- **NAS 경로 공유 원칙** : 주간 영업 현황 보고서는 `00_업무보고/주간보고`에 저장하고 메신저로 NAS 경로를 팀장에게 전달한다.
