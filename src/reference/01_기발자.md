# 개발자 (PM·기획 / 개발자 / QA·테스터) — 업무 PC 파일 정리 규칙

> **적용 환경** : 폐쇄망 · 사내 업무 포털(그룹웨어) · 사내 메신저·메일 파일 첨부 전달 · NAS / SVN 공유 드라이브

---

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 숫자 접두사 정렬 | `00_` → `01_` → `10_` 순으로 탐색기 정렬 고정 |
| 업무보고 최우선 | `00_업무보고`가 항상 최상단 노출 |
| 소스코드 분리 | 소스코드는 사내 SVN/Git, 산출물은 공유드라이브(NAS) |

---

## 파일 전달 흐름

```
메일·메신저 수신 → 01_수신함 임시 보관 → 역할 폴더 이동 + 재명명
→ SVN/Git 커밋 또는 NAS 등록 → 포털 이슈·결재 연동
```

## 운영 규칙

- **`01_수신함`** : 메일·메신저로 받은 파일을 우선 여기에 넣고, 당일 해당 역할 폴더로 이동해 `YYMMDD_내용_출처` 형식으로 재명명한다.
- **NAS 경로 공유 원칙** : 설계서·명세서를 메신저로 공유 요청받을 경우 파일 직접 전달 대신 NAS 경로를 메시지로 전달해 버전 분산을 방지한다.
- **임원회의 자료** : `_초안 → _검토중 → _최종` 상태 태그를 파일명에 붙여 최신 버전을 즉시 식별한다.
- **감리/감사 대응 자료** : 업무 포털 전자결재 완료 즉시 PDF로 내려받아 `14_감리대응`에 이중 보존한다.
- **QA 결함 리포트** : 메일로 PM·고객사에 발송할 때는 NAS에 저장된 파일을 첨부하고, 발송 후 `35_결과보고서`에 발송본을 보관해 수발신 이력을 남긴다.
- **소스코드** : 반드시 사내 SVN/Git에만 보관한다. 로컬에만 존재하는 상태에서 PC 장애 발생 시 전체 유실 위험이 있다.
