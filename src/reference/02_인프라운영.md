# 인프라·운영 — 업무 PC 파일 정리 규칙

> **적용 환경** : 폐쇄망 · 사내 업무 포털(그룹웨어) · 사내 메신저·메일 파일 첨부 전달 · NAS / SVN 공유 드라이브

---

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 숫자 접두사 정렬 | `00_` → `01_` → `20_` 순으로 탐색기 정렬 고정 |
| 업무보고 최우선 | `00_업무보고`가 항상 최상단 노출 |
| 변경요청 번호 부여 | 메일 수신 즉시 ITSM 번호 부여 후 구성관리 폴더 이동 |
| 장애 증적 보관 | 오류 캡처 + 메신저 대화 캡처 모두 증적 폴더에 보관 |

---

## 파일 전달 흐름

```
메일 변경요청 수신 → 01_수신함 임시 보관 → 20_구성관리 이동 + ITSM 번호 부여
→ 포털 변경 승인 결재
```

---

## 폴더 구조

```
📁 인프라_운영
  📁 00_업무보고                           ← 최우선 노출
    📁 주간보고
    📁 월간보고
    📁 임원회의
  📁 01_수신함                             ← 메일 변경요청 첨부본 임시
  📁 20_구성관리
    📄 ITSM-250601_웹서버증설_김운영.xlsx
    📄 ITSM-250610_방화벽변경_이보안.xlsx
  📁 30_장애대응
    📄 INC-032_DB장애_타임라인.hwp
    📁 31_증적                             ← 오류 캡처 + 메신저 대화 캡처
  📁 40_점검결과
  📁 50_매뉴얼
```

---

## 파일명 규칙

| 유형 | 규칙 | 예시 |
|------|------|------|
| 변경관리 | `ITSM-YYMMDD_내용_요청자` | `ITSM-250601_웹서버증설_김운영.xlsx` |
| 장애대응 | `INC-번호_내용_타임라인` | `INC-032_DB장애_타임라인.hwp` |
| 보고서 | `YYMMDD_보고명_상태` | `250603_주간운영보고_확정.hwp` |
| 점검결과 | `YYMMDD_점검유형_결과` | `250531_월간보안점검_완료.xlsx` |

---

## 운영 규칙

- **`01_수신함`** : 메일로 온 변경 요청 첨부파일을 임시 보관하고, 당일 `20_구성관리`로 이동해 ITSM 번호를 파일명에 부여한다.
- **NAS 경로 공유 원칙** : 주간·월간 운영 보고서는 `00_업무보고` 하위에 저장 후 메신저로 NAS 경로를 공유한다. 보고서를 메신저로 직접 전달하면 버전이 분산된다.
- **장애 증적** : 장애 대응 시 메신저로 주고받은 내용도 화면 캡처로 저장해 타임라인 문서와 함께 `31_증적` 폴더에 보관한다.
- **보안 점검 결과** : 취약점 정보가 포함되므로 팀장 접근 전용 NAS 폴더에 별도 저장하고 일반 공유 폴더와 분리한다.
- **변경요청 메일 보존** : 포털 변경 승인 결재 완료 후 결재본 PDF를 내려받아 해당 ITSM 폴더에 이중 보존한다.
